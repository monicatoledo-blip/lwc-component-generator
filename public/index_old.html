📋 Complete Deployment Guide (For Non-Technical Users)

Step 1: Generate Your Component

  1. Click on the "Create Component" tab at the top
  2. Fill out the form with your custom data (name, email, colors, etc.)
  3. Watch the preview update as you type
  4. Click the "⚡ Generate LWC Component" button at the bottom
  5. A ZIP file called unifiedProfileLwc.zip will download to your computer

Step 2: Extract the ZIP File

  1. Find the downloaded unifiedProfileLwc.zip file (usually in your Downloads folder)
  2. Windows: Right-click the file → Select "Extract All" → Click "Extract"
  3. Mac: Double-click the ZIP file (it will automatically extract)
  4. You should now see a folder called unifiedProfileLwc containing 4 files

Step 3: Install Visual Studio Code & Salesforce Extensions

(Skip this step if you already have VS Code with Salesforce extensions installed)

  1. Download and install Visual Studio Code from code.visualstudio.com
  2. Open VS Code
  3. Click the Extensions icon in the left sidebar (looks like 4 squares)
  4. Search for "Salesforce Extension Pack"
  5. Click "Install" on the "Salesforce Extension Pack" by Salesforce
  6. Wait for all extensions to install (this may take a few minutes)

Step 4: Set Up Your Salesforce Project

Option A: If you have an existing Salesforce project

  1. In VS Code, go to File → Open Folder
  2. Navigate to your Salesforce project folder and open it
  3. In the left sidebar, navigate to force-app/main/default/lwc/
  4. Drag and drop the unifiedProfileLwc folder into the lwc folder
  5. Right-click the unifiedProfileLwc folder
  6. Select "SFDX: Deploy Source to Org"
  7. Wait for the success message in the Output panel at the bottom

Option B: If you DON'T have a Salesforce project yet

  1. In VS Code, press Ctrl+Shift+P (Windows) or Cmd+Shift+P (Mac)
  2. Type "SFDX: Create Project" and press Enter
  3. Choose "Standard" as the project template
  4. Enter a project name (e.g., "MyLWCProject") and press Enter
  5. Choose a location to save your project (e.g., Documents folder)
  6. Once created, open the project folder in VS Code
  7. Press Ctrl+Shift+P (Windows) or Cmd+Shift+P (Mac) again
  8. Type "SFDX: Authorize an Org" and press Enter
  9. Choose "Production" or "Sandbox" based on your org type
  10. A browser window will open - log in to your Salesforce org
  11. After successful login, return to VS Code
  12. In the left sidebar, navigate to force-app/main/default/lwc/
  13. Drag and drop the unifiedProfileLwc folder into the lwc folder
  14. Right-click the unifiedProfileLwc folder
  15. Select "SFDX: Deploy Source to Org"
  16. Wait for the success message

Step 5: Add Component to a Lightning Page

  1. Log in to your Salesforce org
  2. Click the gear icon (⚙️) in the top right → Select "Setup"
  3. In the Quick Find box, type "Lightning App Builder" and select it
  4. Click "New" to create a new page, or select an existing page to edit
  5. If creating new: Choose "App Page" or "Record Page", give it a name, click "Next", choose a template, click "Finish"
  6. In the Components panel on the left, scroll down to "Custom" section
  7. You should see "unifiedProfileLwc" in the list
  8. Drag the "unifiedProfileLwc" component onto your page canvas
  9. Click "Save" in the top right
  10. Click "Activate" to make it live
  11. Choose where you want this page to appear (App, App + Record Type, etc.)
  12. Click "Save" again

Need Help?

If you encounter any issues:

  • Make sure you're logged into the correct Salesforce org in VS Code
  • Check that all 4 files are in the unifiedProfileLwc folder
  • Verify your Salesforce user has permission to create Lightning Web Components
  • Try refreshing the Lightning App Builder page if you don't see the component
