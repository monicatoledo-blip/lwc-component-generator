Deployment Instructions

Follow these steps carefully to deploy your custom Lightning Web Component to Salesforce.

Step 1: Generate & Download

  1. Fill out the form above with your custom data
  2. Click Download Component (ZIP)

Step 2: Extract & RENAME (Crucial!)

  1. Extract the downloaded ZIP file:
    • Windows: Right-click → "Extract All"
    • Mac: Double-click the ZIP file
⚠️
STOP: Check the extracted folder name!

If your computer added a number to it like unifiedProfileLwc-1 or unifiedProfileLwc (1), you MUST rename the folder to exactly unifiedProfileLwc.

Salesforce will reject the deployment if the folder name doesn't match the files inside!

Step 3: Move to VS Code

  1. Open your Salesforce project in VS Code
  2. In the left sidebar, expand force-app → main → default → lwc
  3. Drag and drop your renamed unifiedProfileLwc folder directly onto the lwc folder

Step 4: Deploy to Salesforce

  1. Right-click the unifiedProfileLwc folder you just moved
  2. Click SFDX: Deploy Source to Org
  3. Wait for the 'Deploy successful' popup in the bottom right corner

Step 5: Add to Lightning Page

  1. Log into your Salesforce org
  2. Navigate to a Record Page, click the Gear Icon ⚙️, and select Edit Page
  3. Look under the 'Custom' list on the left for Unified Profile Mock and drag it onto the page
  4. Click Save and Activate

Step 6: Fix Broken Images (Salesforce CSP)

Salesforce blocks external images by default. If your Avatar shows a broken image icon:

  1. Go to Salesforce Setup (Gear Icon ⚙️ → Setup)
  2. Search for CSP Trusted Sites and click New Trusted Site
  3. Enter the root domain of your image (e.g., https://image.s4.sfmc-content.com)
  4. Check the box for Allow site for img-src
  5. Click Save and hard-refresh your Lightning page!
💡
Pro Tip: Common image domains to whitelist:
  • https://image.s4.sfmc-content.com - Cumulus Financial assets
  • https://via.placeholder.com - Placeholder images
  • https://i.pravatar.cc - Random avatar generator
